## The Three-Layer System

### Layer 1: DOCTRINE (Non-negotiable)
**Files:** `/docs/`

These are the rules that govern ALL work. They don't change based on trends, client mood, or preferences.

- OPERATING_MANUAL.md — North Star, decision hierarchy, philosophy
- EXECUTION_STANDARDS.md — Tone, length, publishing rules
- BRAND_AUTHORITY.md — How we protect quiet authority
- SEO_LOCATION_STRATEGY.md — Geographic/SEO discipline

**When to use:** Before any decision, check the doctrine first.

---

### Layer 2: STRATEGY (What we're building)
**Files:** `/strategy/`

These are the specific initiatives, plans, and roadmaps.

**Firm Website:**
- CRITICAL_ISSUES.md — Current problems and audit findings
- roadmap.md — How we fix it
- audit.md — Detailed performance breakdown

**Personal Brand (Jason):**
- JASON_TAYLOR_STRATEGY.md — Overview
- content-pillars.md — Deep dive into each pillar
- platforms.md — Per-platform strategy
- 90-day-plan.md — Detailed timeline

---

### Layer 3: EXECUTION (How we do the work)
**Files:** `/content/`, `/team/`, `/tools/`, `/info/`

These are the operational files—templates, workflows, information inventory, asset management.

**Content:**
- CONTENT_REVIEW_CHECKLIST.md — Pre-publish verification
- templates/ — Post templates for each platform
- examples/ — Sample posts (once created)

**Information:**
- WHAT_WE_KNOW.md — Current knowledge inventory
- WHAT_WE_NEED.md — Information gaps
- jason-details.md — Personal details (bikes, recipes, events)
- jason-assets.md — Photos, videos, raw content
- firm-details.md — Offices, services, team

**Team:**
- ROLES.md — Who does what
- WORKFLOWS.md — How work flows
- CONTACTS.md — Team members and escalation

**Tools:**
- INTEGRATIONS.md — Social platforms, scheduling tools
- POSTING_CALENDAR.md — Master content calendar
- ASSET_MANAGEMENT.md — Photo/video organization

## How to Navigate This System

**I need to make a decision:**
1. Check `/docs/` first (doctrine applies)
2. Check relevant `/strategy/` file (is there guidance?)
3. Check `/team/` (is there a workflow?)
4. Make the decision and document it

**I need to create content:**
1. Check `/docs/EXECUTION_STANDARDS.md` (tone/format rules)
2. Check `/strategy/personal-brand/content-pillars.md` (which pillar?)
3. Use template in `/content/templates/`
4. Check `/content/CONTENT_REVIEW_CHECKLIST.md` before publishing

**I need information about Jason:**
1. Check `/info/WHAT_WE_KNOW.md` (is it already documented?)
2. If not, check `/info/WHAT_WE_NEED.md` (do we need it?)
3. If we need it, add to `/info/jason-details.md` or `/info/jason-assets.md`

## Workflow: Information → Content → Publishing

```
1. GATHER INFORMATION
   └─ jason-details.md, jason-assets.md
   └─ What we have? What we're missing?

2. PLAN CONTENT
   └─ POSTING_CALENDAR.md
   └─ Which pillar? Which platform?
   └─ What message? What assets?

3. CREATE CONTENT
   └─ Check doctrine (EXECUTION_STANDARDS.md)
   └─ Use template (templates/)
   └─ Write caption
   └─ Gather/edit assets

4. REVIEW CONTENT
   └─ CONTENT_REVIEW_CHECKLIST.md
   └─ Does it pass all checks?
   └─ Ready to publish?

5. PUBLISH & TRACK
   └─ Post on platform
   └─ Log in POSTING_CALENDAR.md
   └─ Monitor engagement
   └─ Log results in SOCIAL_METRICS.md
```
