## Weekly Content Workflow

### Step 1: Content Capture (Jason)
**Owner:** Jason E. Taylor  
**Frequency:** Weekly (or as events happen)  
**Output:** Raw photos, videos, context notes  

**Process:**
1. Jason captures content throughout the week
2. Sends to social manager with brief context ("riding in mountains," "made escargot")
3. Submits weekly batch on [Day/Time]

---

### Step 2: Asset Organization (Social Manager)
**Owner:** Social Media Manager  
**Frequency:** Upon receipt  
**Input:** Raw photos/videos from Jason  
**Output:** Organized asset folder with metadata  

**Process:**
1. Receive content from Jason
2. Sort by pillar (Rider's Life, Gourmet, Cigars, Wisdom)
3. Organize in `/assets/images/` or `/assets/videos/`
4. Create metadata file with descriptions
5. Log in content calendar

**Turnaround:** 24 hours

---

### Step 3: Content Planning (Social Manager)
**Owner:** Social Media Manager  
**Frequency:** Weekly  
**Input:** Organized assets, content calendar  
**Output:** Planned posts with assigned platforms and dates  

**Process:**
1. Review available assets
2. Check content calendar for pillar balance
3. Decide which pillar × platform for each asset
4. Assign posting dates
5. Create caption draft

**Turnaround:** 24 hours

---

### Step 4: Caption Writing (Social Manager)
**Owner:** Social Media Manager  
**Input:** Asset, pillar, platform decision  
**Output:** Draft caption in Jason's voice  

**Process:**
1. Check EXECUTION_STANDARDS.md for tone rules
2. Check PLATFORMS.md for platform-specific length/style
3. Write caption (no corporate speak)
4. Include hashtags (Instagram)
5. Include CTA if applicable

**Turnaround:** 4 hours per caption

---

### Step 5: Content Review (Reviewer)
**Owner:** Content Reviewer  
**Input:** Draft caption + visual  
**Output:** Approval or revision requests  

**Process:**
1. Read caption without looking at image first
2. Check against EXECUTION_STANDARDS.md (tone match?)
3. Check against BRAND_AUTHORITY.md (authority protected?)
4. Verify information accuracy
5. Check CONTENT_REVIEW_CHECKLIST.md
6. Approve or request revisions

**Turnaround:** 24 hours (target: same day if urgent)

**Approval levels:**
- No changes needed → Approve
- Minor tweaks → Suggest changes inline
- Tone issue → Request rewrite
- Doctrine violation → Reject and explain

---

### Step 6: Publishing (Social Manager)
**Owner:** Social Media Manager  
**Input:** Approved caption + visual  
**Output:** Published post  

**Process:**
1. Format post for platform (Instagram, LinkedIn, Facebook)
2. Schedule or post immediately (per calendar)
3. Log in POSTING_CALENDAR.md
4. Set reminder to monitor engagement

**Timing:**
- Instagram: 4-5 posts/week, Wed-Sat optimal
- LinkedIn: 2-3 posts/week, Tue-Thu morning
- Facebook: 3-4 posts/week, evenings/weekends

---

### Step 7: Engagement (Social Manager)
**Owner:** Social Media Manager  
**Frequency:** Daily  
**Input:** Posted content, comments  
**Output:** Engaged community  

**Process:**
1. Monitor comments hourly (if possible)
2. Respond within 24 hours
3. Reply as Jason, authentically
4. Engage with others' content (shout-outs, likes)
5. Flag inquiries for follow-up

---

### Step 8: Tracking & Reporting (Analytics Lead)
**Owner:** Analytics/Reporting Lead  
**Frequency:** Daily monitoring, weekly summary, monthly deep-dive  
**Input:** Posted content, engagement metrics  
**Output:** Performance reports  

**Process:**
1. Log engagement daily
2. Calculate engagement rate (likes+comments÷followers)
3. Note saves and shares (high-signal metrics)
4. Track DM inquiries
5. Monthly summary report
6. Quarterly strategy review

**Key metrics:**
- Engagement rate (not follower count)
- Save rate (value indicator)
- Share rate (word-of-mouth)
- Inquiry quality
- Conversion to consultation

## Monthly Review Cycle

### Week 1: Planning
- Review last month's performance
- Identify top-performing content types
- Plan pillar balance for coming month
- Brief Jason on upcoming focus areas

### Week 2: Execution
- Execute posting schedule
- Monitor engagement daily
- Respond to comments/DMs
- Capture new content from Jason

### Week 3: Analysis
- Compile performance data
- Identify patterns and insights
- Note community feedback
- Plan adjustments for next month

### Week 4: Strategy Sync
- Full team review meeting
- Discuss metrics and learnings
- Adjust strategy if needed
- Confirm next month's focus

---

## Quarterly Strategy Review

**Frequency:** Every 3 months

**Participants:** All team members + Jason

**Agenda:**
1. Review overall metrics (growth, engagement trends)
2. Assess content pillar balance (are we 40-20-20-20?)
3. Evaluate platform performance
4. Identify what's working (replicate)
5. Identify what's not (discontinue)
6. Plan adjustments for next quarter
7. Adjust doctrine if patterns emerge

**Output:** Updated strategy, documented decisions in git

## Escalation Workflow

**If a question comes up during work:**

```
Question received
    ↓
Does doctrine govern this? → YES → Check /docs/
    ↓ NO
Is it a tone/voice issue? → YES → Ask Reviewer
    ↓ NO
Is it legal/accuracy? → YES → Ask Firm Contact
    ↓ NO
Is it strategic? → YES → Check /strategy/ or escalate
    ↓ NO
Is it technical/platform? → YES → Research or support
    ↓ NO
Document as new question, escalate to team lead
```

---

## Issue Resolution Process

1. **Identify:** What's not working?
2. **Document:** In which file/process?
3. **Propose:** What could fix it?
4. **Test:** Does it work? Does it serve the doctrine?
5. **Implement:** If approved, update workflow
6. **Document:** Commit change to git with explanation

## Communication Standards

### Internal Communication
- **Daily:** Check-in on blockers/progress (async Slack/email okay)
- **Weekly:** Team sync (15 min, or async update)
- **Monthly:** Full review + strategy (1 hour)
- **Quarterly:** Deep-dive + adjustments (2 hours)

### External Communication (with Jason)
- **Frequency:** Weekly content batch + weekly approval check-in
- **Method:** Email, WhatsApp, or agreed channel
- **Turnaround:** 24-48 hours expected

### Documentation Communication
- **Every change:** Git commit with clear message
- **Every decision:** Documented in relevant file
- **Every pattern:** Added to doctrine if it repeats
